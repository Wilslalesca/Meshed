== Context View

The context view provides an overview of the UMA system and its interactions with external entities. It focuses on the system's boundaries, external actors, and key dependencies.

The context diagram provides an overview of the system’s interaction with users and third-party services.

=== System Overview

Our system is an optimised sports practice scheduler designed to organize practices seamlessly with other teams that utilize the same facilities. By quickly finding optimal times that suit everyone’s schedules, our tool allows coaches and athletes to save time, avoid scheduling conflicts, and maximize team practice outcomes by ensuring every session is well attended and productive.

=== External Entities

- **Customers**: End-users of the platform who reserve spaces, make payments, and receive notifications regarding available parking spaces.
- **Parking Staff**: Users who interact with the system in a supervisory or administrative capacity.
- **Municipal Authorities**: Ensure that parking compliance, including proper payment and abiding by time limits.
- **Notification Gateway**: Service that notifies users on parking spot availability.

=== System Interaction

- Student-Athletes, Coaches, and Facility Administrators interact with UMA through a web interface.
- Notifications are sent to external receivers like email or SMS.
