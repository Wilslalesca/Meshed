== Context View

The context view provides an overview of the Meshed system and its interactions with external entities. It focuses on the system's boundaries, external actors, and key dependencies.

The context diagram provides an overview of the system’s interaction with users and third-party services.

=== System Overview

The system is an optimised sports practice scheduler designed to organize practices seamlessly with other teams that utilize the same facilities. By quickly finding optimal times that suit everyone’s schedules, our tool allows coaches and athletes to save time, avoid scheduling conflicts, and maximize team practice outcomes by ensuring every session is well attended and productive.

=== External Entities

- **Users**: (Previously "athletes") Base level users, use the system to upload their schedule and view schedules generated by managers.
- **Managers**: (Previously "coaches") manage teams, generate schedules, book facilities.
- **Facility administrators**: Sipervise facilities, view and handle conflicts
- **Administrators**: Create accounts for users in their organization

=== System Interaction

- Users, Managers, and Facility Administrators interact with Meshed through a web interface.
- Notifications are sent to external receivers like email or SMS.
